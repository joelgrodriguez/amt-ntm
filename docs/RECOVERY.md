# Recovery — after a DevKinsta "pull from environment"

When you pull from Kinsta into DevKinsta, the `wp-content` directory (and likely
this theme folder) gets replaced with Kinsta's older version. This doc gets you
back to working state.

## TL;DR

```bash
# from the theme dir, after the pull wipes it:
cd /Users/jrodriguez/Development/Kinsta/public/newtech/wp-content/themes
rm -rf amt-ntm                                      # only if Kinsta's stale copy is there
git clone https://github.com/joelgrodriguez/amt-ntm.git
cd amt-ntm
git switch dev
npm install
npm run build
```

Then activate the `amt-ntm` theme in WP admin (DevKinsta site). Done.

## Why this works

Everything is on GitHub. `origin/dev` is the source of truth. The DevKinsta
pull cannot touch GitHub, so as long as `dev` is pushed before you pull,
nothing is lost.

## State of the world at last push

- **Repo:** https://github.com/joelgrodriguez/amt-ntm
- **Source-of-truth branch:** `dev`
- **Release branch:** `master` (pushed manually from a master checkout)
- **DevKinsta WP root:** `/Users/jrodriguez/Development/Kinsta/public/newtech`
- **Theme path:** `wp-content/themes/amt-ntm`
- **Node version used locally:** v24.x
- **Build tool:** Vite + Tailwind v4 (`npm run dev` / `npm run build`)

## Step-by-step recovery

### 1. Sanity check before the pull (do this every time)

```bash
cd /Users/jrodriguez/Development/Kinsta/public/newtech/wp-content/themes/amt-ntm
git status                    # working tree should be clean
git log origin/dev..HEAD      # should be empty (nothing unpushed)
git push origin dev           # belt and suspenders
```

If any of those show pending work, **stop and push first**. Don't pull until
this directory has nothing local-only.

### 2. Do the DevKinsta pull

Pull from Kinsta as you normally would. DevKinsta will replace the database
and `wp-content`, including this theme directory.

### 3. Restore the theme

```bash
cd /Users/jrodriguez/Development/Kinsta/public/newtech/wp-content/themes

# If DevKinsta dropped Kinsta's older version here, remove it:
rm -rf amt-ntm

# Clone fresh from GitHub:
git clone https://github.com/joelgrodriguez/amt-ntm.git
cd amt-ntm

# Use the working branch:
git switch dev

# Install JS deps:
npm install

# Build production assets (or `npm run dev` for HMR during work):
npm run build
```

### 4. Re-activate the theme in WP

In WP admin (DevKinsta site), go to **Appearance → Themes** and activate
**amt-ntm**. Kinsta's pull may have left a different theme active.

### 5. Verify

- Front-end loads at the local DevKinsta URL.
- Nav menus appear (Appearance → Menus may need re-assigning to `primary`,
  `mobile`, `footer` locations if Kinsta's DB didn't have them).
- `app/.vite-dev-server` is absent (only present while `npm run dev` runs).

## Worktrees (Shogun)

Shogun worktrees may live at `~/.superset/worktrees/amt-ntm/<branch>/`. Those
are *outside* the WP install, so the DevKinsta pull will not touch them.
They are git worktrees pointing back at this repo's `.git/`.

If this repo's `.git/` gets wiped (full clone), those worktrees will become
orphans pointing at a `.git/` that no longer exists. To clean up:

```bash
# After recovery, from the freshly-cloned theme dir:
git worktree list                  # shows live worktrees
git worktree prune                 # removes stale entries
rm -rf ~/.superset/worktrees/amt-ntm/<dead-branch>   # if directories remain
```

Then re-create worktrees as needed with Shogun.

## Things that survive the pull

- `~/.claude/` — Claude config, skills, memory. Untouched.
- `~/.superset/` — existing worktree storage. Untouched.
- `~/.gitconfig`, SSH keys, etc. — Untouched.
- GitHub itself. Always the safe backup.

## Things that do NOT survive the pull

- Local database content (replaced with Kinsta's).
- `wp-content/uploads` (replaced).
- This theme directory's `.git/` (wiped by directory replacement, depending
  on DevKinsta's behavior). **This is why GitHub must be current first.**
- `node_modules/` and `app/dist/` (rebuild with `npm install && npm run build`).
- `app/.vite-dev-server` (regenerated by `npm run dev`).
- `.shogun/` local worktree notes (rebuild as needed; GitHub issues remain the
  source of truth).
